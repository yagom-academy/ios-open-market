❗ 프로젝트 규칙
 jake가 fork를 따서 각자 브랜치에서 작업 후 통일하여 PR 보내기
► 커밋 규칙
- 최소한 함수마다 commit 하려고 노오오력하기
- 실행 가능한 상태에서 커밋 하기
► 커밋 메시지 규칙
- 제목의 Type은 영문으로 작성
► Type 규칙
- Feat: 새로운 코드나 기능 추가
- Style: 기능이나 로직 변경 없이, 오탈자 등 변경
- Refactor: 변수 이름을 바꾸거나, 코드 수정
- Fix: bug 수정(사용자 입장에서 느껴지는 것들)
► 코딩 컨벤션
- Swift API 디자인 가이드라인을 따기
- 함수는 동사로, 변수나 상수는 명사로
- naming에 신경쓰기
